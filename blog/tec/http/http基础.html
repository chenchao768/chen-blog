    # 网络基础TCP/IP

    tcp/ip协议簇里最重要的就是分层，按层次分为： 应用层 传输层 网络层 链路层

    各个层解释

    • 应用层：决定了向用户提供应用服务时的通讯活动，如FTP,DNS,HTTP协议
    • 传输层： 提供处于网络连接的两台计算机之间的数据传输，如：tcp,udp
    • 网络层：处理网络上流动的数据包，数据包是网络传输的最小单位，该层规定了通过怎样的路径到达对方计算机，并把数据包传送给对方
    • 链路层：处理链接网络的硬件部分

    # TCP三次握手

    三次握手：客户端首先会发送一个带SYN的数据包给对方，服务端在收到后会回传一个带有SYN/ACK标志的数据包已示传达确认信息，最后客户端在回传一个带有ACK标志的数据包，握手结束。

    如果有服务端未响应，客户端会定时发送数据包确认

    # http请求方法

    get: 用来请求访问已被URI识别的资源，指定的资源经服务器解析后返回响应内容。 post: 传输实体主体，主要的母的不是获取响应的主题内容 put: 传输文件，要求在请求报文的主体中包含文件内容，然后保存到请求URI指定的位置。但是由于HTTP/1.1的PUT方法自身不带验证机制，，任何人都可以上传文件，存在安全性问题 ，所以一般web站不使用该方法 head: 和GET一样，只是不返回报文的主体部分，用于确认URI的有效性及资源更新的日期时间等 delete: 删除文件，与PUT相反 options: 询问支持的方法，用于查询针对请求URI指定资源支持的方法

    # 持久链接

    持久连接的特点是，只要任意一端没有明确提出断开连接，则保持TCP连接状态，减少了TCP连接的重复建立与断开所造成的额外开销，减轻服务端的负载

    # cookie解决http无状态协议的问题

    通过服务端返回的set-cookie字段来通知客户端保存Cookie,当再次请求该资源时，请求中会自动加上Cookie值发送

    # 代理

    分为缓存代理和透明代理

    缓存代理： 会预先将资源的副本保存在代理服务器上，当代理再次访问时，就不需要到源服务器去请求 透明代理： 转发请求响应时，不对报文做任何加工的代理类型被称为透明代理

    # 网关

    网关的工作机制和代理十分相似，网关能使通信线路上的服务器提供非HTTP协议服务，利用网关能提高通信的安全性，因为可以在客户端与网关之间的通信线路上加密以确保连接的安全

    # 请求首部字段

    请求首部字段 概念
    Accept 约定返回的内容的格式
    Accept-Charset 通知服务器用户代理支持的字符集以及字符集的相对优先顺序
    Accept-Encoding 告知服务器用户代理支持的内容编码以及编码顺序
    Authorization 该首部字段是用来告知服务器，用户代理的认证信息
    Form 告诉服务器使用用户的电子邮件地址
    Host 请求资源所在的主机以及端口号
    if-match 请求服务器看是否存在对应资源的etag
    if-modified-since 服务器返回的last-modified
    if-none-match 对比服务器返回的e-tag
    if-range 如果指定的if-range字段值和请求资源的Etag值或者时间相一致，则作为范围请求处理，反之，则返回全体资源
    range 向服务器请求部分资源在接收到Range首部字段时，服务器处理后会返回206，无法处理该范围请求时，则会返回200以及全部资源
    referer 告知服务器原始请求的URI
    user-agent 将创建请求的浏览器和用户代理名称等信息传达给服务器

    # 响应报文首部

    响应首部字段 概念
    Accept-Ranges 告知客户端是否能处理范围请求，以指定获取服务端某个部分的资源
    Etag 首部字段Etag能告知客户端实体标识，它是一种可将资源以字符串形式做唯一性标识的方式。服务器会为每份资源分
    location 可以将响应接收方引导至某个与请求URI位置不同的资源
    proxy-authenticate 把由代理服务器所要求的认证信息发送给客户端
    server 告知客户端当前服务器上安装的HTTP服务器应用程序的信息
    WWW-authenticate 告知客户端试用于访问请求URI所指定资源的认证方案和带参数提示的质询，状态码401肯定带有该首部字段
    实体首部字段 概念
    allow 通知客户端能够支持的Request-URI指定资源的所有HTTP方法
    content-encoding 告知客户端服务器对实体的主体部分选用的内容编码方式（gzip,compress,deflate,identity）
    content-language 告知客户端实体主体试用的自然语言
    content-length 表明实体主体部分的大小
    content-location 给出与报文主体部分相对应的URI
    content-md5 由一串MD5算法生成的值，目的在于检查报文主体在传输过程中是否保持完整，以及确认传输到达
    content-range 告知客户端作为响应返回的实体的哪个部分符合范围请求
    content-type 说明了实体主体内 对象的媒体类型
    expires 将资源失效的日期告知客户端
    last-modified 指明资源最终修改时间
