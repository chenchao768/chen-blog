    # 关于csrf攻击以及xss攻击

    csrf攻击：跨站请求伪造（英语：Cross-site request forgery）， 也被称为 one-click attack 或者 session riding，通常缩写为 CSRF 或者 XSRF， 是一种挟制用户在当前已登录的Web应用程序上执行非本意的操作的攻击方法，其根本就是使用用户已登录的用户信息去发起而已请求

    怎么防范CSRF攻击

    1.Get 请求不对数据进行修改
    2.不让第三方网站访问到用户 Cookie
    3.阻止第三方网站请求接口
    4.请求时附带验证信息，比如验证码或者 token

    XSS跨站脚本 XSS指的是攻击者利用漏洞，向 Web 页面中注入恶意代码，当用户浏览该页之时，注入的代码会被执行，从而达到攻击的特殊目的。

    防御方法

    1.浏览器自带防御 （X-XSS-Protection ）HTTP X-XSS-Protection 响应头是 Internet Explorer，Chrome 和 Safari 的一个功能，当检测到跨站脚本攻击(XSS)时，浏览器将停止加载页面。
    2.内容安全策略（Content Security Policy，CSP），实质就是白名单制度，开发者明确告诉客户端，哪些外部资源可以加载和执行，大大增强了网页的安全性。
